
# Walle Ear S3 — 四麦克风声源定位听觉子系统

基于 **ESP32-S3** + **ESP-IDF 5.x** + **4×INMP441** 数字麦克风 + **TinyUSB**，实现机器人听觉前端：实时采集 4 路音频、TDOA 互相关 3D 声源定位、UAC 单声道音频上传、CDC JSON 方向数据上报。

## 特性

| 功能 | 说明 |
|------|------|
| **4 路同步采集** | I2S0（Master）+ I2S1（Slave）共享 BCLK/WS，每路 16kHz / 16-bit PCM |
| **3D 声源定位** | 4 麦非共面布局，6 对 TDOA 互相关 + 3×3 最小二乘求解，无镜像歧义 |
| **UAC 音频流** | ESP32-S3 原生 USB 枚举标准 USB Microphone，单声道 16kHz 16-bit |
| **CDC 数据上报** | 同 USB 口虚拟串口，每秒 1 条 JSON：`{"azi":-45.3,"ele":12.7}` |
| **全中文日志** | ESP_LOG 通过 USB-UART 口输出详细运行状态与调试信息 |
| **零外部芯片** | 仅凭 ESP32-S3 + 4×INMP441，无需额外 ADC/Codec |

## 麦克风阵列布局

4 个 INMP441 采用**非共面**布局，Mic3 离开 Mic0/Mic1/Mic2 所在平面以消除 3D 定位的镜面歧义。

```
              +Y (前方 0°)
               ^
               |
          [Mic2] (0, +50, 100)
               |
               |
[Mic1] --------+-------- [Mic0]
(+30,0,0)      |         (-30,0,0)
               |
               |
          [Mic3] (0, -50, 100)
               |
               v
              -Y (后方)
```

### 坐标表

| 麦克风 | X (mm) | Y (mm) | Z (mm) | 说明 |
|--------|--------|--------|--------|------|
| Mic0 | −30.0 | 0.0 | 0.0 | 底板左前 |
| Mic1 | +30.0 | 0.0 | 0.0 | 底板右前 |
| Mic2 | 0.0 | +50.0 | +100.0 | 前方高处 |
| Mic3 | 0.0 | −50.0 | +100.0 | 后方高处（非共面消歧） |

### 麦克风对（6 对 TDOA）

| 编号 | 对 | 方向向量 (mm) | 基线长度 |
|------|----|--------------|----------|
| P₁ | 0→1 | (+60, 0, 0) | 60.0 |
| P₂ | 0→2 | (+30, +50, +100) | ~115.1 |
| P₃ | 0→3 | (+30, −50, +100) | ~115.1 |
| P₄ | 1→2 | (−30, +50, +100) | ~115.1 |
| P₅ | 1→3 | (−30, −50, +100) | ~115.1 |
| P₆ | 2→3 | (0, −100, 0) | 100.0 |

> 坐标和 Mic 对通过 `main/mic_array_config.h` 中的 `#define` 宏定义，可自由修改。

## 引脚连接

```
              ESP32-S3
         ┌──────────────┐
         │              │
  GPIO4 ─┼─ BCLK ───────┼── 4×INMP441 SCK
  GPIO5 ─┼─ WS ─────────┼── 4×INMP441 WS/LRCLK
  GPIO6 ─┼─ I2S0 DIN ───┼── Mic0 SD (L/R=GND) + Mic1 SD (L/R=3V3)
  GPIO7 ─┼─ I2S1 DIN ───┼── Mic2 SD (L/R=GND) + Mic3 SD (L/R=3V3)
         │              │
   3V3 ──┼──────────────┼── 4×INMP441 VDD
   GND ──┼──────────────┼── 4×INMP441 GND
         │              │
 GPIO19 ─┼─ USB D- ─────┼── 原生 USB (UAC+CDC)
 GPIO20 ─┼─ USB D+ ─────┼── 原生 USB (UAC+CDC)
         └──────────────┘
```

| 信号 | GPIO | 连接对象 |
|------|------|---------|
| BCLK | GPIO4 | 所有 INMP441 SCK |
| WS | GPIO5 | 所有 INMP441 LRCLK |
| I2S0 DIN | GPIO6 | Mic0 SD（L/R = GND 左声道）+ Mic1 SD（L/R = 3V3 右声道） |
| I2S1 DIN | GPIO7 | Mic2 SD（L/R = GND 左声道）+ Mic3 SD（L/R = 3V3 右声道） |
| 3V3 | — | 所有 INMP441 VDD |
| GND | — | 所有 INMP441 GND |
| USB D− | GPIO19 | 原生 USB OTG |
| USB D+ | GPIO20 | 原生 USB OTG |

> **USB 说明**：ESP32-S3 有两个 Type-C 口——**原生 USB 口**（GPIO19/20）枚举 UAC 麦克风 + CDC 虚拟串口，**USB-UART 口**（通常 GPIO43/44）用于烧录、monitor 和 ESP_LOG 调试输出。

## 构建与烧录

### 环境要求

- **ESP-IDF 5.x**（推荐 5.2+）
- Windows / Linux / macOS
- Python 3.8+

### 构建步骤

```powershell
# 1. 进入项目目录
cd F:\well-e-bot\walle_ear_s3

# 2. 设置目标芯片
idf.py set-target esp32s3

# 3. 编译
idf.py build

# 4. 烧录并打开串口监视
idf.py flash monitor
```

### sdkconfig 关键配置

```
CONFIG_IDF_TARGET_ESP32S3=y
CONFIG_SPIRAM=y
CONFIG_SPIRAM_USE_CAPS_ALLOC=y
CONFIG_TINYUSB=y
CONFIG_TINYUSB_AUDIO=y
CONFIG_TINYUSB_CDC=y
```

## 使用方法

### 连接电脑

1. 将 ESP32-S3 的**原生 USB 口**和 **USB-UART 口**都连接到电脑。
2. 原生 USB 口会枚举为一个 **USB Microphone**（UAC）和一个 **串口设备**（CDC）。
3. USB-UART 口用于查看 ESP_LOG 日志（波特率 115200）。

### 查看定位数据

打开任意串口工具（PuTTY、Tera Term、minicom 等），连接 CDC 虚拟串口，波特率无所谓（CDC ACM 自适应），将看到每秒一条 JSON：

```json
{"azi":-45.3,"ele":12.7}
```

无效定位时：

```json
{"azi":null,"ele":null}
```

### 录音测试

电脑端将 USB Microphone 设为默认输入设备，用 Audacity 等软件录制即可听到 Mic0 的实时音频。

```
walle_ear_s3/
├── CMakeLists.txt              # 顶层构建配置
├── partitions.csv              # 分区表
├── sdkconfig.defaults          # 默认 Kconfig 选项
├── README.md                   # 本文件
└── main/
    ├── CMakeLists.txt          # 模块构建配置
    ├── main.c                  # 入口：初始化 + 启动流水线
    ├── i2s_mic_driver.h        # I2S 麦克风驱动头文件（采样率/通道数/位宽宏）
    ├── i2s_mic_driver.c        # I2S0+I2S1 双外设采集、时钟桥接、帧交织
    ├── mic_array_config.h      # 麦克风坐标宏、声速、Mic 对定义
    ├── afe_processor.h         # 定位处理器头文件（结构体、API 声明）
    ├── afe_processor.c         # TDOA 互相关、3D 最小二乘求解、CDC JSON 上报、UAC 音频路径
    ├── usb_composite.h         # TinyUSB 复合设备头文件
    ├── usb_composite.c         # UAC + CDC 枚举与读写
    └── tusb_config.h           # TinyUSB 编译期配置
```

### 模块职责

| 模块 | 职责 |
|------|------|
| **i2s_mic_driver** | I2S0（Master）采集 Mic0/Mic1，I2S1（Slave）采集 Mic2/Mic3；时钟桥接同步两外设；输出交织 4 通道 PCM 帧 |
| **mic_array_config** | 提供 4 麦三维坐标（mm）、声速常量、6 对 Mic 对的静态定义 |
| **afe_processor** | 从 i2s_mic_driver 获取帧 → 6 对互相关 TDOA 估计 → 3×3 最小二乘求解单位方向向量 → 提取 azimuth/elevation；UAC 提取 Mic0 单通道上传；CDC 每秒发送 JSON |
| **usb_composite** | TinyUSB 复合设备：UAC 单声道 16kHz Microphone + CDC ACM 虚拟串口 |

## 配置说明

所有可调参数通过头文件中的宏定义，无需改代码逻辑：

### 麦克风坐标 — `main/mic_array_config.h`

```c
#define MIC_ARRAY_POS0_MM_X -30.0f   // Mic0 X 坐标
#define MIC_ARRAY_POS0_MM_Y   0.0f   // Mic0 Y 坐标
#define MIC_ARRAY_POS0_MM_Z   0.0f   // Mic0 Z 坐标
// ... Mic1/Mic2/Mic3 同理
#define SOUND_SPEED_MM_PER_S 343000  // 声速 (mm/s)
```

### 采样参数 — `main/i2s_mic_driver.h`

```c
#define I2S_MIC_SAMPLE_RATE_HZ 16000    // 采样率
#define I2S_MIC_BITS_PER_SAMPLE 16      // 位宽
#define SAMPLES_PER_CHANNEL 512          // 每通道每帧采样点数
#define MIC_COUNT 4                       // 麦克风数量
```

### 音频源选择 — `main/afe_processor.c`

```c
#define AFE_UAC_AUDIO_SOURCE AFE_UAC_SOURCE_RAW_MIC  // RAW_MIC / TEST_TONE
#define AFE_UAC_RAW_MIC_INDEX 0                        // 上传哪个 Mic（0=Mic0）
```

### TDOA 阈值 — `main/afe_processor.c`

```c
#define AFE_TDOA_MIN_RMS 20.0f          // 最低信号 RMS
#define AFE_TDOA_MIN_CORRELATION 0.25f  // 最低互相关峰值
#define AFE_TDOA_MIN_CONFIDENCE 15      // 最低置信度
```

## 角度定义

### 方位角 (azimuth)

- `+Y` 方向为 **0°**
- 绕原点朝 `+X`（右）旋转为 **正角度**
- 绕原点朝 `−X`（左）旋转为 **负角度**
- 范围：**−180° ~ +180°**

```
          +Y (0°)
           ^
           |
  (-90°) --+-- (+90°)
   -X      |      +X
           |
           v
          -Y (±180°)
```

### 俯仰角 (elevation)

- XY 平面为 **0°**
- `+Z`（上方）为 **正角度**，最大 `+90°`
- `−Z`（下方）为 **负角度**，最小 `−90°`

## 调试建议

### 推荐测试顺序

| 步骤 | 操作 | 验证目标 |
|------|------|---------|
| 1 | 只接 Mic0/Mic1 到 I2S0 | UAC 音频清晰、无杂音 |
| 2 | 切换 `AFE_UAC_RAW_MIC_INDEX` | Mic0 和 Mic1 均可正常上传 |
| 3 | 接入 Mic2/Mic3 到 I2S1 | 日志中 4 通道均有有效波形，I2S1 无 timeout |
| 4 | 查看 mic frame 调试日志 | 4 路 min/max 值合理，Mic2 和 Mic3 相同量级 |
| 5 | CDC 串口观察 JSON 输出 | 定位有效时输出角度，静音时输出 null |
| 6 | 拍手/发声测试 | 方位角随声源位置变化响应正确 |

### 常见问题

| 现象 | 可能原因 | 排查方向 |
|------|---------|---------|
| I2S1 持续 timeout | 时钟未桥接、GPIO7 未连接、Mic2/Mic3 L/R 接错 | 检查 `bridge_i2s_shared_clock()` 日志、万用表测 GPIO7 波形 |
| 4 通道中某路始终为 0 | 该 INMP441 供电异常、L/R 选择错误 | 检查 3V3/GND、L/R 引脚电平 |
| 定位飘忽不定 | 室内反射、麦克风坐标不准、I2S0/I2S1 有固定延迟 | 用吸音材料减少反射、实测校准坐标 |
| CDC 无数据输出 | 原生 USB 口未接或未枚举 | 确认连接的是 GPIO19/20 的原生 USB 口 |
| UAC 有声音但方向始终 null | 信号 RMS 低于阈值 | 增大声源音量或降低 `AFE_TDOA_MIN_RMS` |
