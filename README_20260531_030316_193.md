# Walle Ear S3

> 基于 ESP32-S3 的 4 麦克风阵列机器人听觉前端

## 特性

- **4 路同步采集**：4 颗 INMP441 MEMS 麦克风，16kHz / 16-bit PCM，I2S0 ×2 + I2S1 ×2
- **3D 声源定位**：TDOA 互相关 + 6 对远场最小二乘，输出方位角 (azimuth) 与仰角 (elevation)
- **USB 复合设备**：UAC 标准音频流 + CDC ACM 串口数据上报
- **1Hz 定位上报**：每秒输出一次 JSON 格式的角度信息，便于上位机解析
- **非共面阵列**：4 麦克风不共面布局，从硬件上消除 3D 镜像歧义

## 硬件

| 组件 | 型号 / 规格 |
|------|-------------|
| MCU | ESP32-S3 |
| 麦克风 | 4× INMP441 MEMS 数字麦克风 |
| 接口 | USB 2.0 Full Speed (原生 OTG) |
| 时钟 | I2S0 主模式输出 BCLK/WS，I2S1 从模式共享时钟 |

### 引脚连接

| 信号 | GPIO | 说明 |
|------|------|------|
| BCLK | GPIO4 | I2S0 输出 / I2S1 输入 (共享) |
| WS | GPIO5 | I2S0 输出 / I2S1 输入 (共享) |
| DIN0 | GPIO6 | I2S0 数据输入 (Mic0 L + Mic1 R) |
| DIN1 | GPIO7 | I2S1 数据输入 (Mic2 L + Mic3 R) |

### 麦克风阵列布局

| Mic | X (mm) | Y (mm) | Z (mm) | 位置说明 |
|-----|--------|--------|--------|----------|
| 0 | -30 | 0 | 0 | 左 |
| 1 | 30 | 0 | 0 | 右 |
| 2 | 0 | 50 | 100 | 前上 |
| 3 | 0 | -50 | 100 | 后上 (非共面) |

Mic3 与 Mic0/Mic1/Mic2 不共面，从阵列几何上消除 3D TDOA 的镜像解歧义。

### 角度定义

- **方位角** (azimuth)：+Y 方向为 0°，+X 方向为 +90°，范围 -180° ~ +180°
- **仰角** (elevation)：XY 平面为 0°，+Z 方向为 +90°，范围 -90° ~ +90°

## 构建

```bash
# 设置目标芯片
idf.py set-target esp32s3

# 构建
idf.py build

# 烧录并监控
idf.py flash monitor
```

**要求**：ESP-IDF 5.x，建议启用 PSRAM。

## 使用

USB 连接后，系统枚举两个设备：

| 设备 | 接口 | 说明 |
|------|------|------|
| UAC 麦克风 | USB Audio Class 1.0 | 单声道 16kHz/16-bit PCM (Mic0 raw) |
| CDC ACM 串口 | USB CDC ACM | 每秒输出一行 JSON 定位数据 |

**串口数据格式** (9600-8-N-1，实际波特率由主机 CDC 驱动协商)：

有定位结果时：
```json
{"azi":-45.3,"ele":12.7}
```

无声源时：
```json
{"azi":null,"ele":null}
```

## 配置

| 配置项 | 文件 | 说明 |
|--------|------|------|
| 麦克风坐标 | `main/mic_array_config.h` | `MIC_ARRAY_POS{0-3}_MM_{X,Y,Z}` 宏 |
| 声速 | `main/mic_array_config.h` | `SOUND_SPEED_MM_PER_S` (默认 343000 mm/s) |
| 采样参数 | `main/i2s_mic_driver.h` | `SAMPLES_PER_CHANNEL`、`MIC_COUNT`、采样率 |
| USB 描述符 | `main/tusb_config.h` | TinyUSB 设备描述符与配置 |
| UAC 音频源 | `main/afe_processor.c` | `AFE_UAC_AUDIO_SOURCE` 宏 |

## 调试建议

1. 先接 2 个麦克风，验证 I2S0 + UAC 音频正常
2. 接入全部 4 个麦克风，确认 I2S1 不超时
3. 通过 USB-UART 串口日志查看 4 路 min/max/rms，确认波形有效
4. 开启 TDOA，用拍手等脉冲声源验证角度输出合理
